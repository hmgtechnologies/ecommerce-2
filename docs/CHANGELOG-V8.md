# 🆕 HMG StoreForge v8 — What's New (Detailed)

v8 builds on v1–v7. **Every earlier feature is preserved** — v8 only *adds*. All additions use
**free tools only — no paid AI APIs.**

Headline additions you asked for:
1. **Full multi-language** (English, Nigerian Pidgin, **Hausa, Yoruba, Igbo**).
2. **Offline-first order queue** — place orders with no internet; auto-send when back online.
3. **Vendor split-payout helper** — pay each vendor with bank details from the ledger.
4. **Stronger SEO** so the store is **searchable on Google, Bing, etc.**, and confirmed **installable**.

---

## 1) Full Multi-Language 🌍 (English · Pidgin · Hausa · Yoruba · Igbo)

- **What:** A real translation system (`i18n.js`) with dictionaries for 5 languages and a **language
  dropdown** in the header. Any element marked `data-i18n="key"` is translated; the choice is
  remembered, and `<html lang>` updates for accessibility/SEO.
- **Where:** `store-template/assets/js/i18n.js`; `data-i18n` attributes across `index.html`.
- **Config:** `languages: { enabled: true, default: "en", list: ["en","pcm","ha","yo","ig"] }`.
- **Extensible:** add a language by adding a block to `DICT` in `i18n.js` and an entry to `list`.
- **Why:** reach customers in their first language across Nigeria — more trust, more sales.

---

## 2) Offline-First Order Queue 📴 (free)

- **What:** Combined with the existing PWA service worker, customers can **browse and place orders with
  no internet**. Orders are **queued** locally; an **online/offline bar** shows status. When the
  connection returns, queued orders are **auto-sent** (WhatsApp + Google Sheets logging if configured),
  or the customer taps **Send now**.
- **Where:** `store-template/assets/js/store-extras7.js` (`Queue`, `flush`) + an `#offlineBar` in
  `index.html`; hooks the existing `sf:order` event.
- **Why:** Nigerian connectivity is often patchy — no sale is lost to a dropped connection.

---

## 3) Vendor Split-Payout Helper 💸 (extends `ledger.html`)

- **What:** The payout ledger now has a **💸 Pay** button per vendor that opens WhatsApp with a
  **payout message including the vendor's bank details** (bank, account number, name) and the amount
  owed — making it one tap to instruct/confirm a transfer or Paystack payout.
- **Where:** `store-extras7.js` (`payoutMessage`, `payoutWhatsApp`) + button in `ledger.html`.
- **Config:** add bank fields to each vendor:
  ```js
  { id:"v1", name:"Bella Fashion", whatsapp:"234…", bank:"GTBank", accountNumber:"0123456781", accountName:"Bella Fashion Ltd" }
  ```
- **Why:** completes the marketplace money flow — track commission AND settle vendors, all free.

## 4) Stronger SEO + confirmed installability 🔎📲

- **SEO:** richer head tags (robots index, og:site_name, og:locale, twitter card fields,
  apple-touch-icon, Google verification placeholder), a **multi-page `sitemap.xml`** (now generated by
  **both** the browser generator and the CLI), and `robots.txt` that references the sitemap and blocks
  private pages. Plus the existing JSON-LD `Store`/`Product` data.
- **Guide:** **[`SEO-GUIDE.md`](SEO-GUIDE.md)** — step-by-step submission to **Google Search Console**
  and **Bing Webmaster Tools** (free), rich-results testing, and ranking tips.
- **Installable:** the store (and the generator, from v7) are **PWAs** — installable on phone, laptop,
  and desktop. See **[`INSTALL-AS-APP.md`](INSTALL-AS-APP.md)**.
